# 利用线路间相关性提升预测精度——方案设计说明

> 本文档是对实验结论的延伸设计，探讨如何将**同场站不同输电线路之间的功率相关性**
> 系统地引入预测模型，以进一步降低预测误差。

---

## 一、背景与动机

### 1.1 已有实验结论

根据 `实验结果分析报告.md` 的分析，**加和预测（先预测各子线路，再加和）明显优于直接预测全站**：

| 模型 | 平均 RMSE 改善 | XYB 场站改善（子线路最多） |
|------|--------------|--------------------------|
| Random Forest | −3.52% | −5.01% |
| XGBoost | −2.66% | −3.52% |
| LightGBM | −1.98% | −3.04% |

改善的核心原因是：子线路建模粒度更细，分布更简单，噪声更少。

### 1.2 尚未利用的信息：线路间相关性

当前的加和预测流程中，每条子线路的模型**完全独立训练**，仅使用该线路自身的历史数据作为特征。然而，同一场站的多条线路往往共享相同的风资源场，功率序列之间存在显著的同步性：

- 当甲线路功率下降（风速减弱、限电、故障），乙线路通常也会出现类似变化。
- 反之，一条线路的异常抬升也可以为其他线路的预测提供先验信号。

**如果能将这种线路间相关性显式建模，就有可能在加和预测基础上进一步压缩误差。**

## 二、三种利用方案

### 方案 A：跨线路历史特征增强（推荐，已实现）

#### 核心思想

> 在预测某条线路时，将同场站所有其他线路的**同期历史数据**作为额外特征一并输入模型。

以 XYA 场站（甲线 + 乙线 + 全站）为例：

| 预测目标 | 原始特征（#1 脚本） | 增强特征（#4 脚本） |
|---------|-----------------|-----------------|
| 甲线功率 | 甲线历史 (M+1 步) | 甲线历史 + 乙线历史 |
| 乙线功率 | 乙线历史 (M+1 步) | 乙线历史 + 甲线历史 |
| 全站功率 | 全站历史 (M+1 步) | 全站历史 + 甲线历史 + 乙线历史 |

XYB 场站（丙线 + 丁线 + 戊线 + 全站）类似，特征维度更大，线路间相关性更丰富。

#### 特征维度变化

| 场站 | 原始特征维度（无限电，M=32） | 增强特征维度 |
|------|--------------------------|------------|
| XYA（预测子线路） | (32+1)×1 = 33 | (32+1)×2 = 66 |
| XYB（预测子线路） | (32+1)×1 = 33 | (32+1)×3 = 99 |
| XS（预测子线路） | (32+1)×1 = 33 | (32+1)×2 = 66 |

含限电时各加 32+1=33 维。

#### 优点

- 改动极小，仅修改特征构造逻辑，其余训练流程完全不变
- 所有模型（LightGBM、XGBoost、Ridge、RF、MLP、LSTM）均可直接使用
- LSTM 天然支持多变量时序输入，增强效果预期最显著

#### 预期收益

- 对于树模型：交叉线路特征提供了额外的分裂信号，有望降低 RMSE 1%~5%（基于类似文献）
- 对于线性模型（Ridge）：增益取决于线路间的线性相关强度
- 对于 LSTM：多变量输入是 LSTM 的强项，预期改善最大

```

**运行方式**（与 #1 完全相同）：

```

**输出文件**：

### 方案 B：两阶段堆叠预测（进阶）

#### 核心思想

> 在独立预测各子线路之后，用一个轻量的**元模型（Meta-learner）**将所有子线路预测值融合，替代简单加和操作。

```
第一阶段（已有）：
    甲线历史 → 模型₁ → 甲线预测值 ŷ_甲
    乙线历史 → 模型₂ → 乙线预测值 ŷ_乙

第二阶段（新增）：
    [ŷ_甲, ŷ_乙] → 元模型 → 全站精修预测值 ŷ_station
```

元模型可以是：
- **线性组合**：`ŷ_station = α · ŷ_甲 + β · ŷ_乙`（允许 α ≠ 1、β ≠ 1，修正系统偏差）
- **轻量树模型**：Ridge 或浅层 XGBoost，捕捉非线性融合关系
- **约束优化**：在限电约束下对子线路预测做后处理，修正功率上限

#### 实现要点

- 元模型使用**折外预测（out-of-fold）**对第一阶段模型打分，避免数据泄露
- 训练集用第一阶段的折外预测值训练元模型；测试集先跑完第一阶段，再输入元模型

#### 优点

- 兼容现有流程，只需在 `#3汇总结果分析.py` 之后增加一步
- 可以同时利用多个第一阶段模型的预测（模型集成）
- 在限电场景下，元模型可以学到"当某线限电时另一线如何补偿"的模式

#### 适合场景

- 当各子线路之间存在明显的非线性依赖关系时（如一线限电导致另一线功率非线性上升）
- 当加和策略已优于直接预测，但想进一步优化全站总量精度时

---

### 方案 C：图神经网络（GNN）建模（前沿）

#### 核心思想

> 将场站内各线路抽象为图中的节点，用**图神经网络**对线路间的空间相关性建模，实现端到端的多线路联合预测。

```
图结构示意（XYB 场站）：
    丙线 ─── 丁线
      \      /
       \    /
        戊线
（边权重 = 线路间功率相关系数）
```

实现框架：

- 使用 **PyTorch Geometric** 或 **DGL** 构建图
- 节点特征：各线路历史功率时序
- 边特征：线路间皮尔逊相关系数（可由历史数据计算，或设为可学习参数）
- 模型结构：**ST-GCN（时空图卷积网络）** 或 **Graph WaveNet**

#### 优点

- 自动发现最优的线路间依赖结构
- 可扩展到跨场站预测（将三个场站的线路联合建图）

#### 适合场景

- 数据量充足（样本数 >> 10 万）
- 可接受较高的开发成本和推理延迟
- 已对方案 A/B 做过充分验证，寻求进一步提升空间

## 三、方案对比总结

| 维度 | 方案 A：跨线路特征增强 | 方案 B：两阶段堆叠 | 方案 C：GNN |
|------|----------------------|-----------------|-----------|
| 实现难度 | ⭐（最简单） | ⭐⭐ | ⭐⭐⭐⭐ |
| 代码改动量 | 极小（仅修改特征构造） | 中等（增加元模型训练步骤） | 大（新框架） |
| 对现有流程的侵入性 | 无（独立脚本） | 低（追加步骤） | 高（重构）|
| 预期改善幅度 | 小~中（1%~5%） | 中（2%~8%） | 中~大（但不稳定） |
| 可解释性 | 好（特征重要性可视化） | 中（元模型系数） | 差 |
| 推荐优先级 | ✅ **首选，立即实验** | ✅ 方案 A 后尝试 | 研究方向 |

## 四、从实验结果看最优路径

结合 `实验结果分析报告.md` 的关键发现：

1. **XYB 场站（三线路）加和收益最大**（最高 −5%~−10%），说明线路越多相关性越强，方案 A 在 XYB 的增益预期最显著。

2. **Ridge 在加和策略下收益接近零**，说明 Ridge 已将线路间的线性依赖大部分编码到了自身（全站信号是子线路信号的线性叠加）；方案 A 对 Ridge 增益有限，但对树模型和 LSTM 的增益预期更大。

3. **XGBoost 和 MLP 对含限电敏感**，方案 A 中引入其他线路的历史数据可以帮助模型更好地感知跨线路的限电联动，有望同时改善含限电场景。
