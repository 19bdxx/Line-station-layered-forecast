# 利用线路间相关性提升预测精度——方案设计说明

> 本文档是对实验结论的延伸设计，探讨如何将**同场站不同输电线路之间的功率相关性**
> 系统地引入预测模型，以进一步降低预测误差。

---

## 一、背景与动机

### 1.1 已有实验结论

根据已完成实验的分析，**加和预测（先预测各子线路，再加和）明显优于直接预测全站**，
其核心原因是子线路建模粒度更细、分布更简单、噪声更少。

### 1.2 尚未利用的信息：线路间相关性

当前加和预测流程中，每条子线路的模型**完全独立训练**，仅使用该线路自身的历史数据
作为特征。然而，同一场站的多条线路往往共享相同的风资源场，功率序列之间存在显著的
同步性：

- 当甲线路功率下降（风速减弱、限电、故障），乙线路通常也会出现类似变化。
- 反之，一条线路的异常抬升也可以为其他线路的预测提供先验信号。

**如果能将这种线路间相关性显式建模，就有可能在加和预测基础上进一步压缩误差。**

---

## 二、三种利用方案

### 方案 A：跨线路历史特征增强（推荐，已实现）

#### 核心思想

> 在预测某条线路时，将同场站所有其他线路的**同期历史数据**作为额外特征一并输入模型。

以 XYA 场站（甲线 + 乙线 + 全站）为例：

| 预测目标 | 原始特征（#1 脚本） | 增强特征（#4 脚本） |
|---------|-----------------|-----------------|
| 甲线功率 | 甲线历史 (M+1 步) | 甲线历史 + 乙线历史 + 全站历史 |
| 乙线功率 | 乙线历史 (M+1 步) | 乙线历史 + 甲线历史 + 全站历史 |
| 全站功率 | 全站历史 (M+1 步) | 全站历史 + 甲线历史 + 乙线历史 |

XYB 场站（丙线 + 丁线 + 戊线 + 全站）类似，特征维度更大，线路间相关性更丰富。

#### 特征维度变化（M=32，即 seq_len=33）

| 场站 | 无限电：原始维度 | 无限电：增强维度 | 含限电：原始维度 | 含限电：增强维度 |
|------|--------------|------------|--------------|------------|
| XYA（子线路） | 33×1 = 33 | 33×3 = 99 | 33×2 = 66 | 33×4 = 132 |
| XYB（子线路） | 33×1 = 33 | 33×4 = 132 | 33×2 = 66 | 33×5 = 165 |
| XS（子线路）  | 33×1 = 33 | 33×3 = 99 | 33×2 = 66 | 33×4 = 132 |

#### 优点

- 改动极小，仅修改特征构造逻辑，其余训练流程完全不变
- 所有模型（LightGBM、XGBoost、Ridge、RF、MLP、LSTM）均可直接使用
- LSTM 天然支持多变量时序输入，增强效果预期最显著

#### 预期收益

- 对于树模型：交叉线路特征提供额外分裂信号，有望降低 RMSE 1%~5%
- 对于线性模型（Ridge）：增益取决于线路间的线性相关强度
- 对于 LSTM：多变量输入是 LSTM 的强项，预期改善最大

### 方案 B：两阶段堆叠预测（进阶）

#### 核心思想

> 在独立预测各子线路之后，用一个轻量的**元模型（Meta-learner）**将所有子线路预测值融合，替代简单加和操作。

```
第一阶段（已有）：
    甲线历史 → 模型₁ → 甲线预测值 ŷ_甲
    乙线历史 → 模型₂ → 乙线预测值 ŷ_乙

第二阶段（新增）：
    [ŷ_甲, ŷ_乙] → 元模型 → 全站精修预测值 ŷ_station
```

元模型可以是：
- **线性组合**：`ŷ_station = α · ŷ_甲 + β · ŷ_乙`（修正系统偏差）
- **轻量树模型**：Ridge 或浅层 XGBoost，捕捉非线性融合关系
- **约束优化**：在限电约束下对子线路预测做后处理，修正功率上限

#### 实现要点

- 元模型使用**折外预测（out-of-fold）**对第一阶段模型打分，避免数据泄露
- 训练集用第一阶段的折外预测值训练元模型；测试集先跑完第一阶段，再输入元模型

#### 适合场景

- 当各子线路之间存在明显的非线性依赖关系时（如一线限电导致另一线功率非线性上升）
- 当加和策略已优于直接预测，但想进一步优化全站总量精度时

---

### 方案 C：图神经网络（GNN）建模（前沿）

#### 核心思想

> 将场站内各线路抽象为图中的节点，用**图神经网络**对线路间的空间相关性建模，实现端到端的多线路联合预测。

```
图结构示意（XYB 场站）：
    丙线 ─── 丁线
      \      /
       \    /
        戊线
（边权重 = 线路间功率相关系数）
```

- 使用 **PyTorch Geometric** 或 **DGL** 构建图
- 节点特征：各线路历史功率时序
- 边特征：线路间皮尔逊相关系数
- 模型结构：**ST-GCN（时空图卷积网络）** 或 **Graph WaveNet**

#### 适合场景

- 数据量充足（样本数 >> 10 万）
- 可接受较高的开发成本
- 已对方案 A/B 做过充分验证，寻求进一步提升空间

## 三、方案对比总结

| 维度 | 方案 A：跨线路特征增强 | 方案 B：两阶段堆叠 | 方案 C：GNN |
|------|----------------------|-----------------|-----------|
| 实现难度 | ⭐（最简单） | ⭐⭐ | ⭐⭐⭐⭐ |
| 代码改动量 | 极小（仅修改特征构造） | 中等（增加元模型训练步骤） | 大（新框架） |
| 对现有流程的侵入性 | 无（独立脚本） | 低（追加步骤） | 高（重构）|
| 预期改善幅度 | 小~中（1%~5%） | 中（2%~8%） | 中~大（但不稳定） |
| 可解释性 | 好（特征重要性可视化） | 中（元模型系数） | 差 |
| 推荐优先级 | ✅ **首选，立即实验** | ✅ 方案 A 后尝试 | 研究方向 |

## 四、#4 脚本与 #1完善版 的同步说明

`#4跨线路特征增强训练.py` 已在 `#1开始训练_完善版.py` 的基础上完整同步了
以下所有增强，同时保留跨线路特征构造这一核心差异：

| 改进项 | 说明 |
|--------|------|
| GPU 自动检测与回退 | LightGBM / XGBoost 优先使用 GPU，失败后自动回退 CPU |
| XGBoost 新 API | 使用 `tree_method='hist'` + `device='cuda'/'cpu'` |
| 数据合法性验证 | 训练前检查 X/y 的 NaN/inf；特征构造时逐行过滤非有限值样本 |
| 空样本保护 | 样本量不足时抛出明确错误，不会静默崩溃 |
| Ridge / MLP 增加 SimpleImputer | Pipeline 首步加入中位数填充，兼容少量残余 NaN |
| LSTM hidden_size=128 | 从 64 提升至 128，提升模型容量 |
| LSTM AdamW + weight_decay | 更好的正则化，减少过拟合 |
| LSTM ReduceLROnPlateau | 验证集 loss 不降时自动降低学习率 |
| LSTM AMP 混合精度 | GPU 下启用 torch.amp.autocast + GradScaler，提速约 1.5~2× |
| LSTM 多 worker DataLoader | GPU 训练时 num_workers>0，pin_memory=True，persistent_workers |
| LSTM 完整 loss 合法性校验 | 每个 batch 检查 loss 是否有限，防止 NaN 爆炸 |
| LSTM 早停优化 | patience=15（原20），best_val_loss 容忍 1e-6 抖动 |
| 预测结果校验 | 检查预测数组长度与有限性 |
| RMSE 版本兼容 | 不依赖 `squared=False` 参数，用 `np.sqrt(MSE)` 计算 |
| MAPE 分母修正 | `np.abs(y_test) + 1e-8`（原 `y_test + 1e-8`），正确处理负值 |
| 单实验级别异常捕获 | 内层 try-except，单步失败仅跳过该步，不中断整个模型 |
| 结果 dict 增加 device 字段 | 记录每个实验实际使用的计算设备 |
| 数据加载基础统计 | 加载后打印数据形状、NaN 统计和 inf 总数 |
| 预测步长范围 | `range(1, 97)` 即 t+1 ~ t+96（原 t+1 ~ t+95） |
| `limit_value` 列修正 | 与特征构造同步收集，不再依赖事后按索引重新查询 |
