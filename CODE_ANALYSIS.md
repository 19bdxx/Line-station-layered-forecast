# 仓库代码主要功能分析报告

## 1. 仓库概述

本仓库名为 **Line-station-layered-forecast**（线-站分层预测），是一个针对风电场的**短期功率预测**实验项目。项目采用 **LightGBM** 机器学习模型，对多个风电场及其子线路的有功功率进行多步超前预测，并通过对比"子线路预测加和"与"全站直接预测"两种方法的精度，探究分层预测的效果。

---

## 4. 代码功能逐文件分析

### 4.1 `#1开始训练.py` — 模型训练与预测

#### 核心目标

针对每个场站的每条子线路及全站，在多个预测步长下训练 LightGBM 回归模型，输出预测结果和评估指标。

#### 关键配置

| 参数 | 值 | 含义 |
|---|---|---|
| `M` | 4×60 = 240 | 历史时间窗口（240分钟） |
| `predict_steps` | 15~240（步长15） | 预测超前步长（分钟），共16个 |
| 实验模式 | 含限电 / 无限电 | 是否将限电功率作为输入特征 |

#### 数据流与处理流程

```
原始CSV加载
    ↓
滑动窗口构造特征（历史240分钟功率，可选拼接限电）
    ↓
时序切分 (80% 训练 / 20% 测试)
    ↓
训练集再切分 (80% 训练 / 20% 验证)
    ↓
LightGBM 训练（含 early stopping，最大2000轮）
    ↓
测试集预测
    ↓
指标计算 + 结果保存CSV + RMSE曲线图
```

#### LightGBM 模型参数

| 参数 | 值 |
|---|---|
| `objective` | regression |
| `metric` | rmse |
| `learning_rate` | 0.01 |
| `num_leaves` | 31 |
| `max_depth` | 6 |
| `min_data_in_leaf` | 50 |
| `feature_fraction` | 0.8 |
| `bagging_fraction` | 0.8 |
| `bagging_freq` | 1 |
| `num_boost_round` | 2000（含早停，容忍100轮） |

#### 评估指标

| 指标 | 公式 | 说明 |
|---|---|---|
| RMSE | √(Σ(y_true-y_pred)²/n) | 均方根误差 |
| MAE | Σ|y_true-y_pred|/n | 平均绝对误差 |
| MAPE | Σ|y_true-y_pred|/y_true×100% | 平均绝对百分比误差 |
| R² | 1-SS_res/SS_tot | 决定系数 |
| 偏差率 | 当真值≥0.2×装机容量时：\|pred-true\|/true；否则：\|pred-true\|/(0.2×装机容量) | 分段偏差率（契合电力行业标准） |

#### 输出结构

```
对比实验/
└── XYA/
    ├── XYA_ACTIVE_POWER_JIA_t+15.csv
    ├── XYA_ACTIVE_POWER_JIA_t+15_RMSE_curve.png
    └── ...
对比实验_无限电/
└── XYA/
    └── ...
所有实验汇总_metrics.csv
```

### 4.2 `#2预测结果汇总.py` — 预测结果合并

#### 核心目标

将步骤一生成的大量单文件结果，**按场站+预测步长**进行横向合并，便于后续同步长多线路对比分析。

#### 处理逻辑

```
遍历 对比实验/ 和 对比实验_无限电/ 两个根目录
    ↓
对每个场站目录下的CSV文件：
    解析文件名 → 提取 target_col 和 predict_step
    ↓
字段重命名（避免合并时冲突）：
    y_true  → {子线路名}_true
    y_pred  → {子线路名}_pred
    bias_rate → {子线路名}_bias_rate
    limit_value → {子线路名}_limit_value
    ↓
相同预测步长的多个子线路结果按 timestamp outer join 合并
    ↓
保存到 合并结果/{with_limit|no_limit}/{station}/
```

#### 输出结构

```
合并结果/
├── with_limit/
│   ├── XYA/XYA_t+15_merged.csv
│   ├── XYB/XYB_t+15_merged.csv
│   └── XS/XS_t+15_merged.csv
└── no_limit/
    └── ...
```

每个合并文件包含字段示例（以XYA为例）：

| 字段 | 含义 |
|---|---|
| timestamp | 时间戳 |
| JIA_true | 甲线实测功率 |
| JIA_pred | 甲线预测功率 |
| JIA_bias_rate | 甲线偏差率 |
| YI_true | 乙线实测功率 |
| YI_pred | 乙线预测功率 |
| STATION_true | 全站实测功率 |
| STATION_pred | 全站直接预测功率 |

### 4.3 `#3汇总结果分析.py` — 分层预测 vs 直接预测对比

#### 核心目标

对每个场站在每个预测步长下，定量比较以下两种全站功率预测方案：

- **方案A（直接预测）**：直接对全站功率建模预测（`STATION_pred`）
- **方案B（分层预测加和）**：分别预测各子线路功率，然后加和（`sum_pred = Σ子线路_pred`）

#### 处理逻辑

```
读取 合并结果/ 下所有 *_merged.csv 文件
    ↓
提取全站实测值 STATION_true
提取全站直接预测 STATION_pred
计算子线路预测加和 sum_pred = Σ(子线路_pred)
    ↓
分别计算两种方案的 RMSE、MAE、R²
    ↓
汇总结果保存为 比较分析结果_全站预测加和_vs_直接预测.csv
```

#### 输出字段

| 字段 | 含义 |
|---|---|
| station | 场站名 |
| step | 预测步长（分钟） |
| limit_mode | 含限电/无限电 |
| RMSE_sum | 分层加和预测的RMSE |
| RMSE_direct | 直接预测的RMSE |
| MAE_sum | 分层加和预测的MAE |
| MAE_direct | 直接预测的MAE |
| R2_sum | 分层加和预测的R² |
| R2_direct | 直接预测的R² |

## 6. 关键设计亮点

### 6.1 分层预测思路

项目的核心科学问题是：**"在全站功率预测中，分层预测各条线路后加和，是否优于直接预测全站？"**

这是电力行业集群预测中的典型研究方向，涉及自下而上（bottom-up）预测聚合策略。

### 6.2 限电特征的影响实验

通过"含限电"与"无限电"两组实验的对比，评估限电功率（`LIMIT_POWER`）作为输入特征时对预测精度的影响，具有实际工程意义。

### 6.3 多步超前预测

预测步长从 **t+15分钟** 到 **t+240分钟**（4小时），步长为15分钟，共16个预测时域，覆盖短期到超短期预测范围。

### 6.4 分段偏差率

采用国内电力行业标准化的偏差率计算方法：
- 真实功率 **≥ 0.2 × 装机容量** 时：用真实功率归一化
- 真实功率 **< 0.2 × 装机容量** 时：用 0.2 × 装机容量归一化（避免低功率时分母过小）

### 6.5 早停机制

LightGBM 训练采用早停（Early Stopping，patience=100轮），在验证集 RMSE 不再下降时自动停止，防止过拟合。

## 9. 总结

本仓库实现了一套完整的**风电场功率分层预测实验框架**，涵盖数据处理、模型训练、结果汇总和对比分析四个阶段。核心贡献在于：

1. **系统化对比**了分层预测（线路级 → 加和得全站）与直接全站预测两种策略；
2. **量化了限电特征**对预测精度的影响；
3. **覆盖16个预测时域**，为不同调度需求提供参考；
4. 代码结构清晰，模块化分步执行，易于扩展到更多场站或预测模型。
