# 多模型对比实验说明文档

## 1. 背景与目的

原有实验（脚本 #1～#3）使用 **LightGBM** 对风电场功率进行分层预测，并得出结论：  
**"分层聚合预测（各线路预测值加和）与直接全站预测相比，精度差异具有规律性。"**

本扩展实验的目标是：

> **验证上述结论是否具有模型无关性**——即当我们把 LightGBM 换成其他主流机器学习算法后，加和策略与直接预测策略的相对优劣关系是否保持一致。

---

## 2. 新增脚本总览

| 脚本 | 作用 |
|------|------|
| `CODE/#4多模型对比训练.py` | 在与脚本#1完全相同的数据流程下，支持 5 种模型同时训练并输出预测结果 |
| `CODE/#5多模型结果分析.py` | 读取#4的输出，合并各目标列结果，对比分层加和 vs 直接预测，生成跨模型一致性报告和可视化图 |

---

## 3. 支持的预测模型

| 模型标识 | 算法名称 | 特点 |
|----------|----------|------|
| `lgb` | LightGBM | 与脚本#1相同；梯度提升树，速度快，支持早停 |
| `rf` | 随机森林 (Random Forest) | Bagging 集成，抗过拟合，无需调早停 |
| `xgb` | XGBoost | 梯度提升树，精调能力强，支持早停 |
| `ridge` | 岭回归 (Ridge Regression) | 线性基线模型，训练极快，适合验证线性可分性 |
| `et` | 极端随机树 (Extra Trees) | 随机化更激进，方差更低 |

**设计原则**：各模型均使用相同的特征矩阵（滑动历史窗口展平为特征向量）、相同的训练/测试划分（8:2），以保证公平对比。

---

## 4. 数据流程（与原实验完全一致）

```
原始 CSV（1分钟分辨率）
        │
        ▼
  特征构建：滑动窗口 M=240 步（4小时）
  特征向量 = 历史功率序列（+ 可选限电序列）展平
        │
        ▼
  训练集（前80%）  ──── 验证子集（训练集后20%，供lgb/xgb早停）
  测试集（后20%）
        │
        ▼
  模型训练 → 预测 → 计算 RMSE / MAE / MAPE / R² / 偏差率
        │
        ▼
  输出：多模型实验/{model}/对比实验/{场站}/{目标列}_t+{步长}.csv
```

## 6. 如何运行

### 6.1 安装依赖

```bash
pip install pandas numpy lightgbm scikit-learn xgboost matplotlib
```

### 6.2 配置数据路径

打开 `CODE/#4多模型对比训练.py`，修改第 32 行：

```python
DATA_PATH = r"你的实际数据文件路径/all_stations_输电线未发电置零_1min.csv"
```

### 6.3 选择要运行的模型（可选）

在 `#4多模型对比训练.py` 第 36 行，按需调整：

```python
MODEL_LIST = ['lgb', 'rf', 'xgb', 'ridge', 'et']
# 例如只运行两个模型：
# MODEL_LIST = ['rf', 'xgb']
```

如果只想调整某个模型的超参，修改 `MODEL_PARAMS` 字典（第 55～90 行）即可。

### 6.4 按顺序执行

```bash
# 步骤一：多模型训练（耗时较长，建议在 GPU/多核机器上运行）
python "CODE/#4多模型对比训练.py"

# 步骤二：合并结果并生成跨模型对比报告和图表
python "CODE/#5多模型结果分析.py"
```

> 如果需要同时保留原有 LightGBM 实验结果（脚本#1～#3 的输出），两套实验目录互不干扰：  
> 原实验输出在 `对比实验/` 和 `对比实验_无限电/`；  
> 多模型实验输出在 `多模型实验/`。

## 7. 输出文件说明

### 7.1 `多模型对比_分层vs直接_详细结果.csv`

每行对应一个（模型 × 场站 × 预测步长 × 限电模式）的对比结果：

| 字段 | 说明 |
|------|------|
| `model` | 模型标识（lgb / rf / xgb / ridge / et） |
| `station` | 场站名称（XYA / XYB / XS） |
| `step` | 预测步长（分钟） |
| `limit_mode` | `with_limit`（含限电）或 `no_limit`（不含限电） |
| `RMSE_direct` | 全站直接预测的 RMSE |
| `RMSE_sum` | 子线路加和预测的 RMSE |
| `MAE_direct` / `MAE_sum` | 对应 MAE |
| `R2_direct` / `R2_sum` | 对应 R² |
| `sum_wins_rmse` | 1 表示加和策略 RMSE 更小（更优），0 表示直接预测更优 |

### 7.2 `多模型对比_结论一致性汇总.csv`

| 字段 | 说明 |
|------|------|
| `model` | 模型标识 |
| `total_cases` | 该模型参与比较的总样本数（场站 × 步长 × 限电模式） |
| `sum_wins_count` | 加和策略获胜次数 |
| `sum_win_rate` | 获胜比例（0～1） |
| `conclusion` | 结论标签（≥0.5 视为"加和策略总体更优"） |

**判读方法**：  
- 若各模型的 `sum_win_rate` 均 ≥ 0.5，说明"分层加和优于直接预测"的结论**具有模型无关性**；  
- 若某个模型的 `sum_win_rate` 显著低于其他模型，可能表明该模型本身对全站功率有更强的直接学习能力，值得进一步分析。

### 7.3 RMSE 对比图（`多模型对比_RMSE对比图/`）

每张图包含两个子图，横轴为预测步长（15～240 分钟），纵轴为 RMSE：
- 左图：各模型直接预测 RMSE 随步长的变化曲线
- 右图：各模型分层加和预测 RMSE 随步长的变化曲线

通过对比左右两图中各模型曲线的相对位置，可以直观判断：
1. 各模型在不同步长下的性能排名是否稳定；
2. 加和与直接预测的差距随步长如何变化。

## 8. 各模型特点与注意事项

### LightGBM（`lgb`）
- 本项目的基线模型，已在脚本#1中大量验证；
- 支持基于验证集 RMSE 的早停（最多 2000 轮，早停轮数 100）；
- 对大型特征矩阵（M=240，特征数 241+）表现优秀。

### 随机森林（`rf`）
- 无需验证集早停，直接使用全部训练集（前80%数据）；
- 默认树深 10、每叶最少 20 个样本，防止过拟合；
- 适合特征数较多的宽表数据，但内存占用较高。

### XGBoost（`xgb`）
- 使用验证子集作为早停依据（`early_stopping_rounds=50`）；
- 默认 500 轮迭代、学习率 0.05，比 LightGBM 稍慢但精度接近；
- 可调整 `MODEL_PARAMS['xgb']` 中的超参以进一步优化。

### 岭回归（`ridge`）
- 最简单的线性基线，**训练速度最快**；
- 预期精度低于树模型，但能揭示特征线性相关性；
- 若 Ridge 也支持"加和优于直接"的结论，说明该现象有稳健的统计基础。

### 极端随机树（`et`）
- 在随机森林基础上进一步随机化分裂阈值，偏差略高但方差更低；
- 训练速度通常快于随机森林；
- 默认超参与随机森林相同，可在 `MODEL_PARAMS['et']` 中单独调整。

## 9. 常见问题

**Q：脚本#4运行时间很长，怎么加速？**  
A：可缩短 `MODEL_LIST`（只选 2～3 个模型），或减少 `predict_steps` 的步长数量，或先在单个场站上验证再全量运行。

**Q：`#5多模型结果分析.py` 运行时提示目录不存在？**  
A：请确保先完整运行 `#4多模型对比训练.py`，且 `MODEL_LIST` 中的模型均已训练完成。

**Q：如何添加自定义模型（如 SVR、MLP 等）？**  
A：在 `#4多模型对比训练.py` 的 `build_sklearn_model()` 函数和 `MODEL_PARAMS` 字典中添加新模型配置；  
在 `run_experiment()` 的训练分支（`else` 块）处，新模型会自动被 sklearn 接口处理，无需其他改动。

**Q：如何只重新分析某一个模型的结果？**  
A：在 `#5多模型结果分析.py` 的 `MODEL_LIST` 中只保留目标模型名称，重新运行即可。
