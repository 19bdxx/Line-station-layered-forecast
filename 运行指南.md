# 运行指南：风电场功率分层预测实验

本文档说明如何从零开始，一步一步运行本项目的所有代码，并解释每步的预期输出。

---

## 目录结构

```
Line-station-layered-forecast/
├── RAW_DATA/
│   └── all_stations.csv          # 原始数据（已内置，无需额外下载）
├── CODE/
│   ├── #1开始训练.py              # 步骤一：训练模型，生成预测结果
│   ├── #2预测结果汇总.py           # 步骤二：合并各子文件
│   ├── #3汇总结果分析.py           # 步骤三：对比分析分层预测 vs 直接预测

├── 运行指南.md                    # 本文档

└── run_all.bat                   # Windows 一键运行脚本
```

## 第一步：环境准备

### 1.1 确认 Python 版本

本项目需要 **Python 3.8 或以上版本**。

```bash
python --version
```

### 1.2 安装依赖库

在终端（或命令提示符）中运行：

```bash
pip install pandas numpy lightgbm scikit-learn matplotlib
```

> 如果安装速度慢，可以使用国内镜像：
> ```bash
> pip install pandas numpy lightgbm scikit-learn matplotlib -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 1.3 进入项目根目录

所有脚本都必须从**项目根目录**运行（即 `Line-station-layered-forecast/` 目录下）：

```bash
cd Line-station-layered-forecast
```

## 第二步：运行脚本（按顺序执行）

### ▶ 步骤一：模型训练与预测（`#1开始训练.py`）

**功能：** 对每个场站（XYA、XYB、XS）的每条子线路及全站，在 16 个预测步长（t+15min 到 t+240min）下分别训练 LightGBM 模型，包含"含限电"和"无限电"两种实验模式，共 320 次训练。

**运行命令：**

```bash
python "CODE/#1开始训练.py"
```

**运行时间：** 根据机器性能不同，预计需要 **1~4 小时**（共 320 次模型训练）。

**控制台输出示例：**
```
✅ XYA 含限电 | XYA_ACTIVE_POWER_JIA_t+15
RMSE : 12.34
MAE  : 9.56
MAPE : 8.23%
R²   : 0.9412
```

**生成的文件结构：**
```
对比实验/                             # 含限电实验结果
└── XYA/
    ├── XYA_ACTIVE_POWER_JIA_t+15.csv         # 预测结果
    ├── XYA_ACTIVE_POWER_JIA_t+15_RMSE_curve.png  # 训练曲线图
    ├── XYA_ACTIVE_POWER_JIA_t+15_model.txt       # 保存的模型
    ├── XYA_ACTIVE_POWER_JIA_t+15_feature_importance.csv  # 特征重要性
    └── ...（每个目标列×每个步长各一组）
对比实验_无限电/                       # 无限电实验结果（结构相同）
└── ...
所有实验汇总_metrics.csv              # 所有实验的 RMSE/MAE/MAPE/R² 汇总表
```

**可选配置（在脚本开头修改）：**

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `USE_STAT_FEATURES` | `True` | `True`=统计特征（快速）；`False`=原始滑动窗口（原始方案） |
| `SAVE_MODEL` | `True` | 是否保存训练好的模型文件 |
| `predict_steps` | `[15,30,...,240]` | 预测步长列表（分钟），可按需缩减 |

> **提示：** 如果只想快速验证流程，可以先把 `predict_steps` 改为 `[15, 60]`，只跑2个步长，节省时间。

### ▶ 步骤二：合并预测结果（`#2预测结果汇总.py`）

**前提：** 步骤一已完成，`对比实验/` 和 `对比实验_无限电/` 目录已生成。

**功能：** 将步骤一生成的大量单文件结果，按场站和预测步长横向合并，便于后续同步长多线路对比分析。

**运行命令：**

```bash
python "CODE/#2预测结果汇总.py"
```

**运行时间：** 约 **1~2 分钟**。

**控制台输出示例：**
```
📂 正在处理目录：对比实验 (with_limit)
✅ XYA 场站已完成合并（共 16 个预测步长）
✅ XYB 场站已完成合并（共 16 个预测步长）
✅ XS 场站已完成合并（共 16 个预测步长）

📂 正在处理目录：对比实验_无限电 (no_limit)
✅ XYA 场站已完成合并（共 16 个预测步长）
...
```

**生成的文件结构：**
```
合并结果/
├── with_limit/
│   ├── XYA/
│   │   ├── XYA_t+15_merged.csv     # 包含甲线、乙线、全站在t+15步长的所有预测
│   │   ├── XYA_t+30_merged.csv
│   │   └── ...（共16个文件）
│   ├── XYB/
│   │   └── ...
│   └── XS/
│       └── ...
└── no_limit/
    └── ...（结构相同）
```

**合并文件字段示例（以 XYA_t+15_merged.csv 为例）：**

| 字段 | 含义 |
|---|---|
| `timestamp` | 时间戳 |
| `JIA_true` | 甲线实测功率 |
| `JIA_pred` | 甲线预测功率 |
| `JIA_bias_rate` | 甲线偏差率 |
| `YI_true` | 乙线实测功率 |
| `YI_pred` | 乙线预测功率 |
| `STATION_true` | 全站实测功率 |
| `STATION_pred` | 全站直接预测功率 |

### ▶ 步骤三：对比分析（`#3汇总结果分析.py`）

**前提：** 步骤二已完成，`合并结果/` 目录已生成。

**功能：** 对每个场站、每个预测步长，定量比较两种全站功率预测策略：
- **直接预测**：对全站功率直接建模预测（`STATION_pred`）
- **分层预测加和**：分别预测各子线路，然后累加（`JIA_pred + YI_pred + ...`）

**运行命令：**

```bash
python "CODE/#3汇总结果分析.py"
```

**运行时间：** 约 **10~30 秒**。

**控制台输出示例：**
```
✅ 收集完成: XYA | t+15 | with_limit
✅ 收集完成: XYA | t+30 | with_limit
...
✅ 比较分析完成，结果已保存为：比较分析结果_全站预测加和_vs_直接预测.csv
```

**生成的文件：**
```
比较分析结果_全站预测加和_vs_直接预测.csv
```

**文件字段说明：**

| 字段 | 含义 |
|---|---|
| `station` | 场站名（XYA/XYB/XS） |
| `step` | 预测步长（分钟） |
| `limit_mode` | `with_limit`（含限电）/ `no_limit`（无限电） |
| `RMSE_sum` | 分层加和预测的 RMSE |
| `RMSE_direct` | 直接预测的 RMSE |
| `MAE_sum` | 分层加和预测的 MAE |
| `MAE_direct` | 直接预测的 MAE |
| `R2_sum` | 分层加和预测的 R² |
| `R2_direct` | 直接预测的 R² |

**解读建议：** 比较 `RMSE_sum` 和 `RMSE_direct`。若 `RMSE_sum < RMSE_direct`，说明分层预测加和方案更优。

## 常见问题

**Q：运行第一步时报 `FileNotFoundError`？**
> 确保当前目录在项目根目录（`Line-station-layered-forecast/`），并且 `RAW_DATA/all_stations.csv` 文件存在。

**Q：运行第二步时提示 `目录不存在`？**
> 第二步依赖第一步的输出。请先确保第一步已完整运行。

**Q：如何只跑少量实验以验证流程？**
> 修改 `CODE/#1开始训练.py` 开头的 `predict_steps` 为 `[15, 60]`，并将 `stations` 只保留一个场站，可大幅缩短运行时间。

**Q：运行中出现 `ModuleNotFoundError`？**
> 按照"第一步：环境准备"安装缺失的依赖库。
