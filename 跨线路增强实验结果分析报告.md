## 一、实验背景

### 1.1 两种预测策略

| 策略 | 含义 |
|------|------|
| **加和预测**（sum） | 分别预测每条子线路功率，再加和得到全站功率 |
| **直接预测**（direct） | 以全站功率为目标直接建模 |

### 1.2 两种训练方案

| 方案 | 特征来源 | 代码文件 |
|------|---------|---------|
| **基准方案**（Baseline） | 每条线路仅用自身历史数据 | `#1开始训练_完善版.py` |
| **跨线路增强**（CrossLine） | 每条线路额外引入同场站其他线路的历史数据 | `#4跨线路特征增强训练.py` |

### 1.3 评估指标

- **RMSE**（均方根误差，MW）：越低越好  
- **MAE**（平均绝对误差，MW）：越低越好  
- **R²**（决定系数）：越高越好  
- **加和胜率**：在所有实验配对中，RMSE_sum < RMSE_direct 的比例  
- **ΔRMSE = RMSE_direct − RMSE_sum**：正值 = 加和策略更优

## 七、综合评估与建议

### 7.1 "加和优于直接预测"的结论依然成立

两份实验数据均支持：**子线路分开建模再加和的策略，在绝大多数情况下（67%~76%）优于直接对全站功率建模**，无论使用哪种训练方案。

### 7.2 跨线路增强效果评估

| 模型类型 | 跨线路增强效果 | 原因分析 |
|---------|:----------:|--------|
| **LightGBM** | 轻微负面（整体），短期局部有小收益 | 树模型对冗余/相关特征有一定鲁棒性，但高维特征使树更浅 |
| **Random Forest** | 轻微负面 | 与 LightGBM 类似 |
| **Ridge** | 接近中性 | 线性模型受特征维度影响有限；跨线路特征在线性加权下可能相互抵消 |
| **XGBoost** | 轻微负面 | 与 LightGBM 类似 |
| **MLP** | **显著负面** | 全连接网络对输入维度高度敏感，特征维度从 33 扩至 99~165 时需重新调整网络深度/宽度/正则化 |
| **LSTM** | **严重退化** | 跨线路使输入序列维度 3~4 倍增大，现有超参数（hidden_size、epoch 等）未针对多变量输入调优，导致欠拟合/不收敛 |

### 7.3 后续优化建议

1. **LSTM / MLP 超参数调优**  
   将 `hidden_size` 从 128 扩大至 256~512，增加 LSTM 层数至 2~3 层，对 MLP 增加 Dropout 和 BatchNorm，并针对跨线路高维输入重新搜索学习率和训练轮数。

2. **特征选择 / 降维**  
   跨线路特征中可能存在噪声通道，建议先用 LightGBM feature importance 筛选最相关的跨线路滞后特征，仅保留贡献显著的通道，而非全量引入。

3. **差异化策略**  
   - LightGBM + Random Forest + Ridge：可考虑保留基准方案，跨线路增益不显著  
   - LSTM：优先调优超参数，高维特征带来的上限收益值得追求但需专项调参  
   - MLP：建议增加正则化强度（L2、Dropout）并使用更小的特征子集

4. **步长覆盖**  
   跨线路增强实验目前仅覆盖 t+1~32（8 小时），建议扩展至 t+96（24 小时），以便完整比较长期预测的跨线路效果。
